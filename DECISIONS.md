# Decision Log

> **최종 업데이트:** 2026-03-21
>
> 프로젝트에서 내린 주요 의사결정을 시간순으로 기록하는 문서.
> "왜 이렇게 결정했는가"를 남겨 나중에 재검토할 때 맥락을 파악할 수 있도록 함.

---

## 기록 형식

```
### [날짜] 결정 제목
- **결정:** 무엇을 결정했는가
- **배경:** 왜 이 결정이 필요했는가
- **검토한 옵션:** 어떤 선택지를 검토했는가
- **결정 이유:** 왜 이것을 선택했는가
- **트레이드오프:** 이 결정으로 포기한 것
- **재검토 조건:** 언제 이 결정을 다시 검토할 것인가
```

## 기술 결정

### 2026-03-21 | 프론트엔드 프레임워크: Next.js 선택

- **결정:** Next.js 14 (App Router) + Tailwind CSS
- **배경:** A사 앱 내 웹뷰로 서비스를 제공하는 구조라 모바일 최적화와 빠른 개발이 핵심
- **검토한 옵션:** Vite + React, Next.js, Remix
- **결정 이유:** SSR로 초기 로딩 속도 최적화, API Routes로 백엔드 별도 구성 없이 빠른 프로토타입 가능, Vercel 무료 배포
- **트레이드오프:** React SPA 대비 번들 크기 증가
- **재검토 조건:** 해당 없음 (프로토타입 범위)

---

### 2026-03-21 | LLM 선택: Claude API

- **결정:** Anthropic Claude API (`claude-3-5-sonnet` 예정)
- **배경:** 한국어 금융 텍스트 분석 품질이 핵심 경쟁력
- **검토한 옵션:** OpenAI GPT-4o, Claude API, Google Gemini
- **결정 이유:** 한국어 성능 우수, 긴 컨텍스트 처리(SEC 공시 원문), RAG 적합성
- **트레이드오프:** OpenAI 대비 생태계 라이브러리 적음
- **재검토 조건:** 한국어 성능 비교 테스트 후 GPT-4o가 유의미하게 우수할 경우

---

### 2026-03-21 | 데이터베이스: Supabase 선택

- **결정:** Supabase (PostgreSQL)
- **배경:** 사용자 포트폴리오, 브리핑 히스토리 저장 필요
- **검토한 옵션:** Supabase, PlanetScale, Firebase Firestore
- **결정 이유:** 무료 티어 충분, PostgreSQL 기반으로 관계형 쿼리 가능, RLS(Row Level Security) 지원, 빠른 셋업
- **트레이드오프:** Firebase 대비 실시간 동기화 기능 약함
- **재검토 조건:** 해당 없음 (프로토타입 범위)

---

### 2026-03-21 | 백엔드 구조: Next.js API Routes 통합 (Phase 1)

- **결정:** Phase 1에서는 Next.js API Routes로 백엔드 통합, Phase 2에서 분리 여부 재검토
- **배경:** 프로토타입 속도 vs AI 파이프라인 유연성 사이 트레이드오프
- **검토한 옵션:** Next.js API Routes 통합, FastAPI 분리
- **결정 이유:** Phase 1 MVP는 속도가 우선. RAG 파이프라인 도입 시(Phase 2) Python 생태계 필요 여부 재검토
- **트레이드오프:** Python AI 라이브러리(langchain 등) 사용 불가
- **재검토 조건:** Phase 2 SEC EDGAR RAG 파이프라인 설계 시

---

### 2026-03-21 | 서비스 형태: 독립 웹앱 (프로토타입)

- **결정:** A사 앱 연동 없이 독립 웹앱으로 구현
- **배경:** 채용 과제 프로토타입이라 실제 앱 연동은 불가능
- **검토한 옵션:** 실제 A사 웹뷰 연동, 독립 웹앱
- **결정 이유:** 데모 목적에 집중, 모바일 웹뷰 사이즈(375px)만 맞춤
- **트레이드오프:** 실제 사용자 인증, 실계좌 연동 불가
- **재검토 조건:** 실제 서비스 개발 시

## 프로세스 결정

### 2026-03-21 | Claude Code 개발 도구: Superpowers 글로벌 설치

- **결정:** Superpowers 스킬 프레임워크를 `~/.claude/skills/`에 글로벌 설치
- **배경:** 복잡한 RAG 파이프라인, 멀티소스 Q&A 등 설계가 중요한 기능이 있어 체계적인 개발 워크플로우 필요
- **검토한 옵션:** 글로벌 설치, 프로젝트별 설치, 미설치
- **결정 이유:** 모든 프로젝트에서 brainstorming → writing-plans → executing-plans 워크플로우 활용, Phase 2~3 개발 시 효과 극대화
- **트레이드오프:** 모든 Claude Code 세션에서 자동 트리거됨 (원하지 않을 때도)
- **재검토 조건:** 불필요하게 개입이 잦을 경우 프로젝트 로컬로 이전
