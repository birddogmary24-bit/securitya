- **재검토 조건:** 언제 이 결정을 다시 검토할 것인가

- **트레이드오프:** Vercel 서버리스 함수 활용 못 함. 별도 스크립트 유지 필요
- **재검토 조건:** Vercel Pro 전환 시 (300초 제한)

### 2026-03-22 | Cron 타이밍: 06:30 KST (Finnhub+AI분석) / 07:30 KST (SEC)

- **결정:** Finnhub 수집 + AI 분석 생성을 06:30 KST, SEC 수집을 07:30 KST에 실행
- **배경:** 사용자가 출근 전(07:00~08:00)에 브리핑을 확인하므로 그 전에 데이터 준비 필요
- **검토한 옵션:** 06:00 KST (미장 마감 직후), 09:00 KST (출근 후), 06:30 KST (출근 전 여유)
- **결정 이유:** 미장 16:00 ET = 06:00 KST 마감 직후 30분 여유를 두고 수집 시작. AI 분석 생성까지 포함하여 07:00 전 완료 목표
- **트레이드오프:** 애프터마켓(16:00~20:00 ET) 이벤트는 반영 불가
- **재검토 조건:** 사용자 브리핑 열람 패턴 데이터 확보 후

### 2026-03-22 | 종목 Tier 재설계: 3-Tier + ETF 분리

- **결정:** Tier 1 (50종목, 전체수집) + Tier 2 (100 개별주식, 전체수집) + Tier 3 (~400 개별주식+ETF, quote+news)
- **배경:** 기존 1,000종목 3단계(50/200/750)에서 API 호출량 최적화 + ETF는 재무데이터 불필요
- **검토한 옵션:** 기존 유지 (50/200/750), 축소+ETF분리 (50/100/400), 2-Tier 단순화
- **결정 이유:**
  - Tier 1+2는 개별주식만 포함하여 재무/애널리스트/목표가 등 전체 수집
  - ETF는 `isEtf` 플래그로 분리 → SEC 공시 수집 제외 (ETF는 SEC 공시 구조 다름)
  - Tier 3는 quote+news만 수집하여 API 호출량 절약
- **트레이드오프:** 커버리지 1,000→550으로 축소
- **재검토 조건:** 사용자 종목 검색 패턴 분석 후 Tier 3 확장

### 2026-03-22 | Tier 3 On-demand 수집

- **결정:** Tier 1/2/3에 없는 종목은 사용자 검색 시 Finnhub에서 즉시 수집 후 DB 캐싱 (24h TTL)
- **배경:** 550종목으로 커버리지 축소됐지만, 사용자가 임의 종목 검색 가능해야 함
- **검토한 옵션:** 전체 종목 사전 수집, On-demand만, Tier 1/2 + On-demand
- **결정 이유:** 사전 수집은 API 호출량 과다. On-demand로 필요 시에만 수집하고 7일 내 활성 Tier 3 종목은 배치에도 포함
- **트레이드오프:** 첫 검색 시 응답 지연 (Finnhub API 호출)
- **재검토 조건:** 활성 사용자 증가 시 인기 종목 자동 Tier 승급 로직 검토

### 2026-03-22 | 캐시 아키텍처: 브리핑 캐시 → 종목별 AI 분석 사전 생성 캐시

- **결정:** briefing_cache(요청별 전체 브리핑 캐시) → stock_analysis_cache(종목별 사전 생성) + market_overview_cache(시장 전체) 구조로 변경
- **배경:** briefing_cache는 포트폴리오+페르소나 조합별로 캐시 → 캐시 히트율 낮음. 종목별로 사전 생성하면 어떤 포트폴리오 조합이든 캐시 조합 가능
- **검토한 옵션:** briefing_cache 유지 (요청별), 종목별 사전 생성 캐시, 하이브리드
- **결정 이유:**
  - Cron(06:30 KST)에서 Finnhub 수집 직후 Tier 1+2 종목별 AI 분석 사전 생성
  - 브리핑 요청 시 해당 포트폴리오 종목의 캐시된 분석을 조합하여 즉시 반환
  - 캐시 히트율 극대화: 종목 A의 분석은 A를 보유한 모든 사용자가 공유
- **트레이드오프:** Cron 실행 시간 증가 (AI 분석 생성 포함), briefing_cache 레거시화
- **재검토 조건:** 사용자 수 증가 시 캐시 TTL/갱신 전략 조정
